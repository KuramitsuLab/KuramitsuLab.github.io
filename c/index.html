#2. 入出力

オンラインジャッジシステムは、 入力に対する出力がマッチするかで正答を判定する。 まず、標準入出力を正しく使いこなすことが重要となる。

#2.1. 標準入出力とは

プログラムには、データの入力(Input)と出力(Output)がある。

入力は、ユーザがキーボードから入力すること場合もあるし、ファイルから読み込む場合もある。 同じく、出力もコンソール画面に表示される場合もあるし、ファイルに保存される場合もある。

どちらの場合でも、データは単なる文字ストリーム（文字列）とみなせる。 そこで、入力元や出力先を気にすることなく、入出力を扱えるようにしたものが、 標準入力(stdin)と標準出力(stdout)である。 接頭語のstd は、標準(standard)から来ている。

#2.2. 標準出力

標準出力は、コンソール画面に表示させるのと同義である。

まず、C言語のhello,worldを思い出してみよう。 これは、標準出力の典型的なプログラム例であった。 

#include<stdio.h>

int main()
{
  printf("Hello World\n");
  return 0;
}

ちなみに、最後の"\n"は改行を意味する特別な文字である。 (“\n”を入れないと改行されない。)

#2.2.1. CからC++へ

C++ は、C言語と高い互換性があり、 C言語のソースコードでもそのまま実行できることが多い。

まず、可能な限りC風に書きたい場合は、#include <cstdio>する。 すると、printf() や scanf() などのCおなじみの入出力関数が利用できるようになる。 

#include<cstdio>

int main()
{
  std::printf("hello,world\n");
  return 0;
}

printf()の前についたstd:: は、名前空間と呼ばれる。 プログラムが大規模になると、同じ名前の関数が衝突することを避けるために、 このように名前空間を関数名やグローバル変数名の前に付けて区別する。 ただし、プログラムが小規模な間は、関数ごとに名前空間を書くのは不便である。

普段は、次のようにusing namespace std を宣言して、省略可能にしておく。 

#include<cstdio>
using namespace std;

int main()
{
  printf("Hello World\n");
  return 0;
}

変数の値や計算結果を表示するときは、%d や %f の書式(format)を使う。 

#include<cstdio>
using namespace std;

int main()
{
  int a = 1;
  double b = 1.0;
  printf("%d %f\n", a, a+b);
  return 0;
}

書式は、型によって異なる。 つまり、printf()を使う場合は、書式と型の対応を覚えておかなければならない。 C++は、書式と型の対応を気にしなくても出力できるようになっている。

#2.2.2. よりC++らしいiostream

標準入出力をデータストリームとして扱う iostream を使うと、 型を気にすることなく、標準出力(cout)に 直接的に変数の値や計算結果を出力できるようになる。 

#include <iostream>
using namespace std;

int main()
{
  int a = 1;
  double b = 1.0;
  cout << a;
  cout << " ";     // 空白
  cout << (a+b);
  cout << endl;    // 改行
  return 0;
}

もちろん、cout の記法を習得した方がよい。

Note. より詳しい解説は教科書「2.2.4 コードの書き方(p.31)」を参照のこと。

#2.2.3. print関数 (Python3)

Python3 は、print() 関数を用いて標準出力に出力できる。 

print("Hello World")

C言語のprintf()と異なり、改行コード"\n"を追加する必要はない。 Python の print()は、自動的に改行を追加してくれる。

変数の値や計算結果の出力も引数を加えると、空白を間に挿入して出力してくれる。 

a = 1
b = 1.0
print(a, a+b)

変数の型を気にしなくてよいので便利である。 空白(sep)や改行(end)を取り除くときは、次のようにオプションを追加する。 

print(a, a+b, sep='')   # 空白なし
print(a, a+b, end='')   # 改行なし
print(a, a+b, sep='', end='')   # 空白改行なし

#2.3. 標準入力

標準入力は、C/C++ も Python3 も少々複雑である。 入力される文字ストリームを扱うデータの単位を理解する必要がある。

  • 行 - 改行コード(“\n”) で区切られた文字列　(例. “1 2”)
  • 字句 - 空白（もしくは改行）で区切られた文字列 (例. “1”)

#2.3.1. 入力ストリーム cin (C++14)

C言語 は、scanf()関数を用いて、変数にデータを読み込む。 

  int a;
  scanf("%d", &a);

もちろん、C++でも #include<cstdio>すれば、scanf()を使うことができる。 しかし、標準入力と同じく、iostream を使うと、書式を気にすることなく、 cin から値を読み込むことができる。 

#include <iostream>
using namespace std;

int main()
{
  int a, b;
  double c;
  a << cin;
  b << cin;
  c << cin;  // double 型で値を読む
  ...
}

変数の型にあわせて自動的に変換してくれるので便利である。 読み込みは字句単位となり、改行は空白と区別されることなく字句区切りとなる。 字句単位はわかりやすいので何回か書けば慣れるだろう。

#2.3.2. input関数 (Python3)

Python3 は、input() を用いて標準入力から読み込む。 

s = input()

注意する点は、input()は字句ではなく、行単位の読み込みになる。 次のように読み込むと、s には文字列して"1 2"が読み込まれる。

プログラマは、字句単位に分割して、整数に変換する必要がある。 これは「決まり文句」があって、次のように高階関数mapを使って書くと良い． 

a, b = map(int, input().split())

高階関数は、Pythonの中でも難度の高い記法なので、 ここでは「決まり文句」として覚えておこう。（後日、解説する。）

注意しなければならないのは、 1行にいくつ字句があるかによって書き方を変える必要がある点である。 １行に字句がひとつしかないときは分割の必要はない。 そのような場合は、単にint()で変換するだけで良い。 

a     = int(input())    # 1行に1つのとき
a,b   = map(int, input().split())  #2つのとき
a,b,c = map(int, input().split())  #3つのとき

整数値以外のデータを読むこともときどき発生する。 その場合は、次の通り、字句からの変換を変える

  • 浮動小数点数として読む場合は、int の代わりに float を使う.
  • 文字列として読む場合は、int の代わりに str を使う.

#2.5. AOJよくある間違い

AOJ は慣れないうちは、なかなかACは取れない。 以下のよくある間違いをヒントにしてACを取ろう。

#2.5.1. PE(Presentation Error)

AOJ は Output (出力) と 一字一句同じにように出力する必要がある。

(対処法) Output をよく読んで同じように出力する。

確認項目: 改行忘れ (例. ITP1_1_C/Rectangle)

1行というのは、改行まで出力して1行となる。

Output. 面積と周の長さを１つの空白で区切って１行に出力して下さい 

  printf("%d %d", area, len);    /* (誤) 改行がない */
  printf("%d %d\n", area, len);  /* (正) */
確認項目: 空白忘れ (例. ITP1_2_A)

出力は、空白の数もあっていなければなりません。

Output a より b の方が大きければ a < b a より b の方が小さければ、 a > b a と b が等しければ、 a == b 

  printf("a<b\n");    /* (誤) 空白がありませんよ */
  printf("a < b\n");  /* (正) */

#2.5.2. WA(Wrong Answer)

簡単にいえば、AOJ側のJudgeデータと答えが一致しないケースといえる。 ここでは、手元(含む, Paiza, IDEONE)ではInput/Outputと同じ得られるけど、 AOJ からAccepted されない場合の対処法を書きます。

確認項目: 入力を最後まで読んでいますか？

（対処方法）AOJは、１回の実行で複数のデータを読み込んで複数回問題を解くことが多い。 １回しか解いてない場合は、２回目以降の判定でマッチしない。 もう一度、問題文をよく読んでみる。

確認項目: 制約条件(Constraints)をみる

問題文の Input/Output はあくまでも問題の理解を助けるためのサンプル。 判定データは制約条件を満たす別のデータが使われる。

Constraints 1 ≤ a ≤ 10,000 データセットの数は 10,000 を超えない。

問題文中の制約条件を読んで正しく動作するようにプログラムしてみよう。

#3. 制御構造

プログラミング初期の最大の発明は、構造化定理である。 つまり、順次、条件分岐、反復の三種類の制御構造を組み合わせれば、 どのようなプログラムも書くことできる。

#3.1. 順次(ブロック)

順次とは、上から順番にプログラムを実行する構造である。 順番に実行するプログラムの単位を文と呼び、文のまとまりをブロックと呼ぶ。 ブロックは、0個以上の文からなる。 ブロックの中が0個の文を空文と呼ぶ。1つの文は、単文と呼ぶ。 単文の場合は、ブロックを省略することもできる。

ブロック自体は、文の一種として解釈される。 ブロックの中に文の代わりにブロックを入れることで、 入れ子状にプログラムをネストすることができる。 これによって、自由自在に制御構造を組み合わせられるようになっている。

C++14

C++ は、Cと同じく中括弧{ } によってブロックを書く。 次の例は、ブロックの中に文があり、 その中に更にネストされてブロックがある構造である。 

  {
    <文1>;
    <文2>;
    {
      <文3>;
    }
    <文4>;
    <文5>;
  }

ブロックは、ネストされることが多い。 文がどのレベルのブロックに属しているか理解しやすいように、 インデント（字下げ）するようにする。

Python3

Python3 は、: 記号でブロックが始まり、 インデント（字下げ）の深さが同じ間はブロックが続くことを意味する。 次は、ブロックの中に、更にネストされてブロックがある構造である。 

  <ブロックの始まり>  :
    <文1>
    <文2>
    <ブロックの始まり> :
      <文3>
    <文4>
    <文5>

Python は、C/C++ と異なり、正しくインデントを揃えなければならない。 空白文字は、全角空白は使ってならない。 空白とTABを混ぜると混乱の元になる。 どちらかの文字に統一してインデントするのがよい。

Python の空文

Python は、インデントによるブロック記法であるため、 0個の文から空文を書くことができない。 何もしないことを意味する空文を書きたいときは、 次のとおり pass という特別な文を用いる。 

  pass

#3.2. 条件分岐

条件分岐は、条件式によって処理内容を切り替える基本的な構造である。 いくつか書き方があるが、if文を確実に使いこなせば良い。

C++

C++のif文は次のように書く。ブロックは、{ } で範囲を指定する。 読みやすさのため、同じブロックの文は同じ深さのインデントで書くのがよい。 

  if(<条件式>) {
    // 条件式が真(true)のとき実行するブロック
  }
  else {
    // 条件式が偽(false)のとき実行するブロック
  }

条件が複数あるときは、else if のように else 節にif文を書く。 

  if(<条件式>) {
    // ＜条件式＞が真(true)のとき実行するブロック
  }
  else if(<条件式２>) {
    // ＜条件式２＞が真(true)のとき実行するブロック    
  }
  else {
    // 条件式が全て偽(false)のとき実行するブロック
  }

Note. 詳しくは、C++14 教科書「4.2.1 if文」(p.77)も参照

Python3

Python は、C/C++ と似ているが、: でブロックを始めるところが異なる。 ブロックは、同じ深さのインデントが続く限り同じブロックと解釈される。 

  if <条件式>: 
    # 条件式が真(true)のとき実行するブロック
  else:
    # 条件式が偽(false)のとき実行するブロック

条件が複数あるときは、elif のようにして続ける。 

  if <条件式>: 
    # ＜条件式＞が真(true)のとき実行するブロック
  elif <条件式２>: 
    # ＜条件式２＞が真(true)のとき実行するブロック
  else:
    # 条件式が全て偽(false)のとき実行するブロック
例題
  • Range ヒント: 複数の条件式は、論理演算子で合成する。
  • Small, Large, or Equal elseブロックにif文を入れ子にする、いわゆる「else if (elif)」を使う

#3.3. N回反復する

もうひとつの頻出パターンは、「N回反復する」である。 Nは、定数でも、プログラムの計算結果でもよい。 これも頻出パターンなので覚えておこう。

C++
  for(int i = 0; i < N; i++) {
    ... // 反復するブロック
    ...
  }
Python3
  for i in range(0, N):   # Nは含まれない
    ... ## 反復するブロック
    ...

あとから登場する配列（リスト）でバグを少なくするため、 「N回反復する」パターンは 0 からN-1までカウントするようにする。

例題
  • Many Hello World 1000回コピペしてはいけません

#3.4. while & break

反復(iteration)は、繰り返しやループとも呼ばれる構造である。 while 文や for文など、色々な文法がる。 しかし、文法よりも頻出パターンで覚えるのがよい。

「while & break」 は、最も簡単な反復の書き方で頻出パターンといえる。 反復したいブロックをwhile文で囲み、 ブロック内から条件を満たしたら break(強制的にループを抜ける）する。

C++
  while(true) {
    ...
    if(<条件式>) break;  // 条件が成り立ったとき、反復を抜ける
    ...
  }
Python3
  while True:
    ...
    if <条件式>: break  ## 条件が成り立ったとき、反復を抜ける
    ...

注意：この書き方は、多少危険である。条件式で正しくbreak できなければ、 無限ループ、つまりプログラムが停止しないバグに陥りやすい。 プログラムを実行する前に必ず break で停止することを確認しよう。

  • Print a Rectangle ヒント：２重ループを用いる

#3.5. 演習問題

#4. 数値表現

コンピュータ上の数値表現と演算子を理解して正しく使えるようにしましょう。

#4.1. 論理値

論理値は、真(true)、偽(false) を表す2値である。bool 型で型付けされている。 コード上では次のリテラルで表現される。

C++14
  true
  false
Python3

Python は英大文字で始まるので注意が必要. 

  True
  False

論理値の代わりに、整数値の1 と 0 で代用することもあるが、 整数と論理値は概念が違うので混在させて利用しない方がよい。

#4.1.1. 論理式

論理式とは、bool 型で型付けされた式、つまり評価すると論理値がえられる式である。 例えば、x < y のような比較は、論理式となる。

#4.1.2. 論理演算

論理演算子は、論理積(&&, and), 論理和(||, or), 論理否定(!, not) に相当する演算子. 論理式を合成するときに利用する.

P,Qを論理式とすると：

C++14
  P && Q          // 論理積
  P || Q          // 論理和
  !P              // 否定
Python3
  P and Q         // 論理積
  P or Q          // 論理和
  not P           // 否定

#4.2. 整数

整数は、コンピュータ上で表現される値の中でもっとも基本的な値である。

C/C++の整数は、コンピュータ内の2進数の表現と対応しており、 8ビット,16ビット, 32ビット, 64ビットの処理が可能である。

  • char – 8ビット整数
  • int – 32ビット整数 (最大値の目安 )
  • long int – 32ビット(64ビットの場合もある)
  • long long int – 64ビット整数 最大値の目安 ()

通常は、int型を利用し、 最大値がintで桁溢れしそうな場合は long long intを使うとよい。 ちなみに、ビット長の桁溢れのことを整数オーバーフロー(integer overflow)と呼ぶ。

Python の整数は、無限長の整数である。桁数に制限はない。 プログラムは、桁溢れを心配する必要がないが、CPUの演算能力を直接使えないため、 演算性能はあまりよくない。

#4.2.1. 整数演算

演算子 C++ Python3 例
加算 1 + 2 1 + 2 3
減算 1 - 2 1 - 2 -1
乗算 2 * 3 2 * 3 6
徐算 7 / 2 7 // 2 3
7 / 2 3.5
余算 7 % 5 7 % 5 2
累乗 2 ** 8 256
絶対値 abs(-1) abs(-1) 1

  • abs(),max(),min() 関数を使うときは、#include<algorithm> する
Python3の注意
  • 徐算 /は、浮動小数点数の結果になる。 C/C++同様、整数値で切り捨てたいときは//で割る。
練習 (★はちょっと難しい)
  • Min,Max,Sum ヒント：値の最大値に注意して適切な型を選ぶこと
  • べき乗★ ヒント: C++の場合は (a * b) % M = ((a % M) * (b % M)) % Mで桁溢れを回避

#4.3. 浮動小数点数

コンピュータ上では、整数と実数（浮動小数点数と呼ぶ）は厳密に区別される。 この区別は、C++もPython3も同じである。

浮動小数点数は、整数と区別するため、 明示的に0.0 のように小数点以下記述する。 もしくは、e もしくは E を含めることで指数表現することもできる。 

  整数  0   1   256   0xff
  浮動小数点数   0.0   1.0    3.14159  10e-9

浮動小数点数は、IEEE754規格にもとづく、コンピュータ内の小数点数の表現である。 C/C++において、次の2種類の型が用意されている。

  • float – 32ビット浮動小数点数 (精度が足りない)
  • double – 64ビット浮動小数点数 (こちらを使う)

Python3 では、浮動小数点数の型はfloatである。 名前はfloatであるが、内部ではdoubleが使われており精度は十分である。

#4.3.1. 型強制

整数と浮動小数点数を演算すると、自動的に整数が浮動小数点数に変換されて 計算される。これを型強制という。

#4.3.2. 整数との変換

整数と浮動小数点数の変換は、次のとおり行う。

C++の場合
  int(1.2)   // 1.2をintに(切り捨て)
  double(1)  // 整数をdoubleに
Python3の場合
  int(1.2)   # 1.2をintに(切り捨て)
  float(1)   # 整数をfloatに
